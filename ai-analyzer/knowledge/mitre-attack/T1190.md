# T1190 - Exploit Public-Facing Application

## MITRE ATT&CK 技术说明

**战术**: Initial Access (初始访问)
**技术**: 利用面向公众的应用程序

## 描述

攻击者利用面向互联网的 Web 应用程序中的漏洞来获取初始访问权限。这是外部攻击者进入内网最常见的入口点。

## 常见攻击手法与 Payload 特征

| 攻击类型 | 典型 Payload | 编码绕过变体 |
|---------|-------------|-------------|
| SQL 注入 | `id=1 UNION SELECT 1,2,3` | `id=1+UNION/**/SELECT`、`id=1%20UNION%2F%2A%2A%2FSELECT` |
| 目录遍历 | `../../../etc/passwd` | `%252e%252e%252f`、`....//....//etc/passwd` |
| RCE 命令注入 | `;cat /etc/passwd` | `;c%61t%20/etc/passwd`、`;ca''t /etc/passwd` |
| 文件包含 | `?file=/etc/passwd` | `php://filter/convert.base64-encode/resource=index` |
| SSTI | `${7*7}` | `#{7*7}`、`{{7*7}}`、`*{7*7}` |
| XXE | `<!DOCTYPE foo [<!ENTITY xxe SYSTEM "file:///etc/passwd">]>` | UTF-16 编码实体 |
| SSRF | `url=http://169.254.169.254/latest/meta-data/` | `url=gopher://127.0.0.1:6379/_FLUSHALL` |
| Webshell 上传 | `<?php @eval($_POST['cmd']);?>` | `<?=`$_GET[0]`?>`、Base64 混淆 |

## 成功判定

| 指标 | 成功特征 | 失败特征 |
|------|---------|---------|
| HTTP 状态码 | 200、302（重定向到攻击者控制页面） | 403、404、500 |
| 响应体大小 | 异常大（返回数据库内容）或异常小（盲注） | 正常错误页面 |
| 响应内容 | 包含 `root:x:0:0:`（文件读取）、数据库表结构 | Web 应用默认错误页 |
| 后续流量 | 同源 IP 短时间内出现 webshell 访问、反弹 shell 连接 | 无后续异常 |

## 关联分析重点

- **同 community_id**：查看同一 TCP 会话中的请求和响应，判断 payload 是否到达目标
- **同源 IP ±300s**：攻击者通常在注入成功后立即上传 webshell 或建立反弹连接
- **同目的 IP 历史**：若目标 IP 近期有多个不同攻击类型的告警，说明正在被持续攻击
