# T1068 - Exploitation for Privilege Escalation

## MITRE ATT&CK 技术说明

**战术**: Privilege Escalation (权限提升) / Exploitation
**技术**: 漏洞利用提权

## 描述

攻击者利用软件或操作系统中的漏洞提升权限或获取代码执行能力。

## 高危漏洞 Payload 特征

| 漏洞 | Payload 关键特征 | 检测要点 |
|------|-----------------|---------|
| Log4Shell (CVE-2021-44228) | `${jndi:ldap://`、`${jndi:rmi://` | Header/Body/URL 均可能携带，需全量检测 |
| Struts2 OGNL | `%{`、`${`、`#a=@java.lang.Runtime` | URL 参数或 Content-Type 中注入 |
| Spring4Shell | `class.module.classLoader` | 请求参数链 manipulation |
| Fastjson 反序列化 | `@type`、`com.sun.rowset.JdbcRowSetImpl` | JSON Body 中的类型注入 |
| Shiro 反序列化 | `rememberMe=deleteMe` | Cookie 中 Base64 序列化数据 |
| WebLogic | `/_async/AsyncResponseService`、`com.bea.console.handles.HandleFactory` | T3/IIOP 协议或 HTTP 路径特征 |
| Confluence SSTI | `/${@java.lang.Runtime@getRuntime().exec(` | URL 路径中的表达式注入 |
| Tomcat | `/examples/jsp/%2e%2e/` | 路径遍历绕过 |

## 成功判定

- **RCE 成功**：响应中包含命令执行结果（如 `uid=0(root)`），或目标主机出现出站连接
- **反序列化成功**：目标主机连接到攻击者控制的 LDAP/RMI 服务器
- **SSTI 成功**：响应体中出现表达式计算结果（如 `${7*7}` 返回 `49`）
- **后续流量**：同源 IP 或同目的 IP 出现反弹 shell、webshell 访问、C2 通信

## 关联分析重点

- **同 community_id**：完整的请求-响应，判断 payload 是否触发
- **利用后出站连接**：漏洞利用成功后，目标 IP 通常会出现到攻击者 IP 的新连接
- **DNS 外带**：利用成功后可能出现 DNS 查询携带敏感数据
