# T1055 - Process Injection

## MITRE ATT&CK 技术说明

**战术**: Defense Evasion (防御规避) / Privilege Escalation
**技术**: 进程注入

## 描述

攻击者将恶意代码注入到合法进程中，以规避检测、提权或持久化。在网络层面表现为异常的进程网络行为。

## 网络可见的攻击特征

| 攻击手法 | 网络特征 | 说明 |
|---------|---------|------|
| Mimikatz 凭据转储 | 内网主机异常的 SMB/Kerberos 流量 | LSASS 进程被注入后产生异常认证流量 |
| 进程注入后 C2 | 合法进程（如 explorer.exe）发起网络连接 | 进程名与网络行为不匹配 |
| 持久化 C2 | 固定间隔的出站连接 | 计划任务触发的 C2 心跳 |
| DLL 注入 | 加载远程 DLL 的网络请求 | `\\server\share\malicious.dll` 类路径 |

## 成功判定

- **凭据窃取成功**：内网出现异常的 Kerberos 票据请求或 SMB 登录
- **注入后 C2**：合法进程（非浏览器/更新程序）发起出站 HTTP/HTTPS
- **横向移动前兆**：凭据窃取后短时间内出现内网横向连接

## 关联分析重点

- **内网认证流量**：Kerberos（88端口）、SMB（445端口）、LDAP（389端口）异常
- **进程网络行为**：网络层可观察到端口和频率，进程层需主机日志
- **时间线**：凭据窃取 → 横向移动的时间间隔通常在分钟级
