# T1071 - Application Layer Protocol

## MITRE ATT&CK 技术说明

**战术**: Command and Control (命令与控制)
**技术**: 应用层协议通信

## 描述

攻击者使用现有应用层协议（HTTP/HTTPS/DNS等）与 C2 服务器通信，以混入正常流量。

## C2 通信特征

| 协议 | 检测特征 | 典型工具 |
|------|---------|---------|
| HTTP | 固定间隔请求、请求路径规律化、无 Referer、UA 异常 | Cobalt Strike、Metasploit |
| HTTPS | TLS SNI 与证书不匹配、自签名证书、JA3 指纹异常 | 各种 RAT |
| DNS | 超长子域名、TXT 记录异常、高频查询同一域名 | dnscat2、iodine、Cobalt Strike DNS Beacon |
| ICMP | ICMP payload 携带数据、异常大的 ICMP 包 | ICMP 隧道工具 |

## 成功判定

- **C2 已建立**：目标主机持续与固定 IP/域名通信，间隔规律（心跳）
- **Beacon 行为**：固定间隔（如每 60s）的 HTTP 请求，响应体大小相似
- **数据外泄**：出站流量异常增大，或 DNS 查询中携带 Base64 编码数据
- **DGA 域名**：域名无明显语义、长度异常、TLD 使用异常

## 关联分析重点

- **同目的 IP 出站流量**：目标主机在告警时间前后的所有出站连接
- **DNS 查询历史**：同源 IP 的 DNS 查询模式（DGA 检测、域名轮换）
- **TLS SNI 交叉验证**：SNI 与 DNS 解析结果是否一致
- **流量频率统计**：是否出现固定间隔的心跳式通信
