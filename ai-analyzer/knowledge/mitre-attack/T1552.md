# T1552 - Unsecured Credentials

## MITRE ATT&CK 技术说明

**战术**: Credential Access (凭据窃取)
**技术**: 凭据泄露

## 描述

攻击者从配置文件、版本控制仓库、环境变量等不安全的位置搜索凭据。

## 常见泄露路径与特征

| 泄露类型 | 典型请求路径 | 检测要点 |
|---------|------------|---------|
| Git 仓库 | `/.git/config`、`/.git/HEAD` | 访问 `.git` 目录 |
| 环境变量 | `/.env`、`/.env.local`、`/.env.production` | 访问环境配置文件 |
| 配置文件 | `/config.php`、`/wp-config.php`、`/application.yml` | 访问应用配置 |
| 备份文件 | `/backup.sql`、`/db.bak`、`/web.zip` | 访问备份文件 |
| 密钥文件 | `/.ssh/id_rsa`、`/.aws/credentials` | 访问密钥目录 |
| 源码泄露 | `/.svn/`、`/.DS_Store`、`/WEB-INF/web.xml` | 版本控制或部署信息 |
| 接口文档 | `/swagger-ui.html`、`/actuator/env` | 框架调试接口暴露 |

## 成功判定

- **HTTP 200**：配置文件存在且被读取（响应体包含密钥、密码等敏感内容）
- **响应体特征**：包含 `password=`、`secret=`、`api_key=`、`AKIA`（AWS Key 前缀）
- **后续利用**：泄露后短时间内出现使用该凭据的登录或 API 调用

## 关联分析重点

- **同源 IP 请求序列**：攻击者通常批量探测多个泄露路径
- **响应状态码分布**：大量 404 中穿插 200，命中泄露文件
- **同目的 IP 历史**：目标应用是否在其他路径也被探测
