# T1572 - Protocol Tunneling

## MITRE ATT&CK 技术说明

**战术**: Command and Control (命令与控制)
**技术**: 协议隧道

## 描述

攻击者将一个协议封装在另一个协议中传输，绕过网络访问控制或混淆通信内容。

## 隧道类型与检测

| 隧道类型 | 特征 | 常见工具 |
|---------|------|---------|
| DNS 隧道 | 超长子域名、TXT/A 记录异常、高频查询 | dnscat2、iodine、dns2tcp |
| ICMP 隧道 | ICMP payload 非标准、包大小异常 | ptunnel、icmpsh |
| HTTP 隧道 | HTTP body 携带二进制数据、固定路径 | reGeorg、suo5、Neo-reGeorg |
| SSH 隧道 | SSH 连接后大量数据传输、端口转发 | ssh -L/-R/-D |
| 云隧道 | Cloudflare Tunnel、ngrok 等反向代理 | cloudflared、ngrok |

## 成功判定

- **DNS 隧道**：同一域名查询频率远超正常（如 >100 次/分钟），子域名长度 >50 字符
- **HTTP 隧道**：固定 URL 路径的 POST 请求，请求/响应体大小相似（心跳式）
- **SSH 隧道**：SSH 会话持续时间长，传输数据量远超交互式 Shell

## 关联分析重点

- **DNS 统计**：同源 IP 的 DNS 查询频率、子域名长度分布、TXT 记录比例
- **流量基线**：与该 IP 的历史通信模式对比（频率、大小、方向）
- **协议异常**：非标准端口上的协议行为（如 53 端口的非 DNS 流量）
