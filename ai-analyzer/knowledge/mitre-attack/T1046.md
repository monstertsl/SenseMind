# T1046 - Network Service Discovery

## MITRE ATT&CK 技术说明

**战术**: Discovery (发现)
**技术**: 网络服务扫描

## 描述

攻击者扫描网络以发现可用的服务、开放端口和运行中的应用程序，为后续攻击做准备。

## 工具指纹

| 工具 | User-Agent 特征 | 行为特征 |
|------|----------------|---------|
| nmap | `Nmap Scripting Engine` | SYN 扫描，端口递增 |
| masscan | 无固定 UA 或 `masscan/1.0` | 超高速全端口扫描 |
| nuclei | `nuclei - open-source project` | 模板化漏洞检测，URL 特征明显 |
| nikto | `Nikto` | Web 漏洞扫描，大量探测请求 |
| gobuster/dirb | 无固定 UA | 目录爆破，404 比例高 |
| sqlmap | `sqlmap/1.x` | SQL 注入自动化，含 `--` 等测试标记 |
| AWVS/Burp | `Acunetix`、`Burp Suite` | 商业扫描器，请求模式规律 |

## 威胁评估

| 扫描类型 | 威胁等级 | 说明 |
|---------|---------|------|
| 全端口扫描 | 高 | 攻击者做全面侦察，可能紧跟漏洞利用 |
| Web 目录扫描 | 中 | 寻找管理后台、配置文件等入口 |
| 指定端口扫描 | 中 | 针对性探测特定服务（如 22、3389、445） |
| 慢速扫描 | 高 | 规避检测的低频扫描，需长时间关联才能发现 |

## 关联分析重点

- **扫描→利用时间关联**：扫描结束后短时间内同源 IP 出现漏洞利用告警
- **同源 IP 历史**：该 IP 是否在多个目标上有扫描行为
- **目的端口分布**：扫描的端口范围和重点端口反映攻击意图
