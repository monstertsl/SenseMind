# SOC 处置手册: 11_execution (命令与脚本执行)

## 适用告警

PowerShell、cmd.exe、Bash、Shell 执行、宏脚本、Java 调用
MITRE T1059

## 分析要点

### 判断是否误报
- **正常运维**：管理员通过 SSH 执行管理命令
- **CI/CD 流水线**：Jenkins/GitLab CI 的脚本执行
- **应用后台**：应用本身的命令调用（如调用 ffmpeg/imagemagick）
- **安全脚本**：安全扫描工具的命令执行

### 判断攻击是否成功
- **命令执行**：HTTP 响应包含命令输出（`uid=`、`whoami` 结果、目录列表）
- **反弹 Shell**：目标主机向外部 IP 发起反向连接（常见端口 4444、8080、53）
- **文件下载**：命令执行后出现 `wget`/`curl`/`certutil` 下载行为
- **Webshell 执行**：异常 URL 路径接收命令参数并返回结果

### 命令注入变体
| 变体 | 示例 | 绕过方式 |
|------|------|---------|
| 基本注入 | `;cat /etc/passwd` | 分号分隔 |
| 管道注入 | `\|id` | 管道符 |
| 命令替换 | `$(whoami)` | $() 替换 |
| 引号绕过 | `ca''t /etc/passwd` | 引号分割 |
| 编码绕过 | `c%61t /etc/passwd` | URL 编码 |

### 关联分析
- **同 community_id**：查看命令注入的请求和响应
- **同源 IP 后续**：命令执行成功后通常紧跟文件下载或反弹连接
- **同目的 IP 出站**：目标主机是否在告警后发起新连接

## 处置流程

1. **紧急**：确认命令执行成功 → 立即隔离主机、排查后门和反弹 Shell
2. **高**：命令注入尝试 → 紧急修补漏洞、封禁 IP
3. **中**：可疑命令执行 → 监控、确认是否为运维操作
4. **低**：误报（CI/CD/运维） → 白名单
