# SOC 处置手册: 05_c2 (恶意通信与C2)

## 适用告警

木马、C2 Beacon、反向 Shell、DGA 域名、Cobalt Strike、Metasploit 通信
MITRE T1071

## 分析要点

### 判断是否误报
- **合法远程管理**：TeamViewer、AnyDesk、向日葵等合法远程工具
- **云服务**：连接到已知云服务（AWS/Azure/阿里云）的正常 API 调用
- **更新服务**：Windows Update、apt update 等系统更新流量
- **CDN 流量**：Cloudflare/CloudFront 等 CDN 的正常回源

### 判断威胁程度
| 指标 | 确认威胁 | 高度可疑 |
|------|---------|---------|
| 域名 | 命中威胁情报 IOC | 未知域名 + 新注册 |
| IP | 已知 C2 IP | Tor 出口节点、IDC IP |
| 通信模式 | 心跳间隔固定（如每 60s） | 请求路径规律化 |
| DNS | DGA 特征（无语义、长域名） | DNS 查询频率异常 |
| TLS | SNI 与证书不匹配 | 自签名证书、JA3 异常 |

### 关联分析
- **同目的 IP 出站流量**：目标主机在告警前后的所有出站连接（找 C2 心跳）
- **DNS 查询历史**：同源 IP 的 DNS 查询模式（DGA 检测、域名轮换）
- **流量频率**：是否出现固定间隔的规律性请求
- **历史告警**：该主机是否曾触发漏洞利用或暴破告警（攻击链关联）

## 处置流程

1. **紧急**：确认 C2 通信 → 立即隔离主机、启动应急响应、全量排查
2. **高**：可疑 C2 通信 → 隔离主机、深入分析、封禁域名/IP
3. **中**：可疑域名 → 封禁域名、持续监控
4. **低**：误报（合法服务） → 白名单
