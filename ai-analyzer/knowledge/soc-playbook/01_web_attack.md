# SOC 处置手册: 01_web_attack (Web应用攻击)

## 适用告警

SQL 注入、XSS、目录遍历、文件包含、RCE、XXE、SSTI、SSRF、命令注入等
MITRE T1190

## 分析要点

### 判断是否误报
- **扫描器特征**：User-Agent 含 `sqlmap`、`nuclei`、`nikto`、`nmap`、`Acunetix`、`Burp`
- **授权测试**：源 IP 是否为安全团队/渗透测试 IP 段
- **正常业务**：请求路径是否为业务正常接口，参数值是否为合法业务数据
- **WAF 探测**：部分告警为 WAF 自身规则验证流量

### 判断攻击是否成功
| 响应码 | 含义 | 判断 |
|-------|------|------|
| 200 | 请求成功 | 检查响应体大小是否异常大（SQLi 返回数据）或包含敏感内容 |
| 302 | 重定向 | 可能是攻击成功后重定向到攻击者页面 |
| 403/404 | 拒绝/不存在 | 攻击可能失败（payload 被 WAF 拦截或路径不存在） |
| 500 | 服务器错误 | 可能触发漏洞但执行异常，需进一步确认 |

- **文件读取成功**：响应体包含 `root:x:0:0:` 等 passwd 文件内容
- **SQLi 成功**：响应体大小与正常请求差异大，或返回数据库表结构
- **RCE 成功**：响应体包含命令执行输出（`uid=`、`Linux`、`Windows`）
- **后续 webshell**：同目的 IP 出现异常 URL 访问（短路径、无扩展名、参数传命令）

### 关联分析
- **同 community_id**：完整请求-响应链，判断 payload 是否到达目标
- **同源 IP ±300s**：扫描器通常批量发送多个攻击 payload
- **同目的 IP 历史**：目标是否持续被攻击（多源攻击同一目标）

## 处置流程

1. **紧急**：确认 RCE/SQLi 成功 → 立即隔离目标主机、检查 webshell、排查数据泄露
2. **高**：利用尝试但未确认 → 紧急修补漏洞、更新 WAF 规则、封禁源 IP
3. **中**：扫描器批量探测未成功 → 封禁 IP、监控后续行为
4. **低**：误报（授权扫描/正常业务） → 白名单
