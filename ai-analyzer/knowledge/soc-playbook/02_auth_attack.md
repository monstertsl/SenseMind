# SOC 处置手册: 02_auth_attack (身份认证攻击)

## 适用告警

SSH/RDP/FTP/HTTP 暴力破解、密码喷洒、凭据填充、账户枚举
MITRE T1110

## 分析要点

### 判断是否误报
- **运维批量操作**：Ansible/SaltStack/Puppet 等配置管理工具的批量 SSH 登录
- **内网可信 IP**：源 IP 为堡垒机、运维网段
- **监控探测**：Zabbix/Prometheus 等监控系统的登录探测
- **正常用户忘记密码**：少量失败后成功，且 IP 为用户常用 IP

### 判断攻击是否成功
- **失败→成功转换**：同一源 IP 对同一账户多次失败后出现成功（时间窗口内）
- **成功后异常**：登录后立即创建新用户、修改配置、下载文件
- **登录时间异常**：成功登录发生在非工作时间（凌晨、周末）
- **多账户成功**：同一源 IP 成功登录多个账户（密码喷洒特征）

### 关联分析
- **同源 IP 频率**：统计短时间内的登录尝试次数和目标账户数
- **同源 IP 历史**：该 IP 是否有其他攻击行为（扫描→暴破→利用链）
- **成功后流量**：登录成功后是否出现横向移动、数据外传

## 处置流程

1. **紧急**：确认成功登录 → 立即禁用账户、隔离主机、排查后门
2. **高**：大量失败尝试 → 封禁 IP、启用账户锁定、强制密码重置
3. **中**：少量尝试 → 监控、加强密码策略
4. **低**：误报（运维/监控） → 白名单
