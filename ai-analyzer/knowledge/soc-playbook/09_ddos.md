# SOC 处置手册: 09_ddos (DDoS攻击)

## 适用告警

SYN Flood、UDP Flood、HTTP Flood、Slowloris、CC 攻击、反射放大
MITRE T1498

## 分析要点

### 判断是否误报
- **正常流量突增**：营销活动、系统上线导致的流量高峰
- **健康检查**：LB/监控系统的健康探测流量
- **CDN 回源**：CDN 节点回源流量集中
- **爬虫抓取**：搜索引擎爬虫的高频抓取

### 判断攻击类型
| 类型 | 特征 | 目标层 |
|------|------|-------|
| SYN Flood | 大量 SYN 无 ACK，半连接数高 | 网络层 |
| UDP Flood | 高频 UDP，目标端口分散 | 网络层 |
| HTTP Flood | 高频 HTTP，请求模式相似 | 应用层 |
| CC 攻击 | 针对单一 URL 高频请求 | 应用层 |
| Slowloris | 慢速发送 HTTP 头，保持连接 | 应用层 |
| 反射放大 | 源端口 53/123，响应远大于请求 | 网络层 |

### 判断攻击影响
- **服务不可用**：目标 IP 连接数突增，正常请求超时
- **带宽耗尽**：入站流量较基线突增数倍
- **应用过载**：HTTP 502/503 错误率升高，响应时间变长

### 关联分析
- **流量突增时间**：与告警时间戳对比确认攻击窗口
- **源 IP 分布**：DDoS 多源 IP，单源可能是 DoS 或测试
- **目的端口/路径**：判断攻击目标是网络层还是应用层

## 处置流程

1. **紧急**：服务不可用 → 启动 DDoS 清洗/CDN 切换、联系运营商
2. **高**：大流量攻击 → 上游限速、WAF 规则拦截、扩容
3. **中**：小规模攻击 → 限速、监控
4. **低**：误报（营销/爬虫） → 白名单
