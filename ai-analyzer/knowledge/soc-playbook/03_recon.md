# SOC 处置手册: 03_recon (扫描探测行为)

## 适用告警

端口扫描、目录扫描、服务发现、漏洞扫描器指纹
MITRE T1046

## 分析要点

### 判断是否误报
- **授权安全评估**：源 IP 为安全团队或渗透测试服务商
- **运维监控**：Zabbix/Nagios 等监控系统的端口探测
- **CDN/搜索爬虫**：Google/Bing/百度爬虫的 URL 探测
- **内网资产发现**：内网安全工具的自动发现（如堡垒机扫描存活主机）

### 判断威胁程度
| 扫描类型 | 威胁等级 | 判断依据 |
|---------|---------|---------|
| 全端口扫描 | 高 | 探测所有端口，意图全面侦察 |
| 指定端口扫描 | 中 | 针对 22/3389/445/3306 等高危端口 |
| 目录爆破 | 中 | 404 比例高，探测管理后台/配置文件 |
| 漏洞扫描器 | 高 | nuclei/AWVS 等工具，扫描后可能紧跟利用 |
| 慢速扫描 | 高 | 低频扫描规避检测，需长时间关联 |

### 关联分析
- **扫描→利用时间关联**：扫描结束后短时间内同源 IP 出现漏洞利用告警
- **同源 IP 历史**：该 IP 在多个目标上的扫描行为（横向侦察）
- **扫描范围**：目的 IP 数量和端口分布反映攻击规模

## 处置流程

1. **高**：外网全端口/漏洞扫描 → 封禁 IP、检查是否有后续利用
2. **中**：目录/服务扫描 → 封禁 IP、更新 WAF 规则、检查暴露面
3. **低**：内网扫描 → 确认是否为授权操作
4. **低**：误报（爬虫/监控） → 白名单
