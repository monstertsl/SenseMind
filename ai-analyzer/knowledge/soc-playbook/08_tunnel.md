# SOC 处置手册: 08_tunnel (隧道通信)

## 适用告警

DNS 隧道、ICMP 隧道、HTTP 隧道、SSH 隧道、反向代理隧道
MITRE T1572

## 分析要点

### 判断是否误报
- **正常 DNS**：DNS 查询频率正常、子域名长度正常、知名 DNS 服务商
- **合法 VPN**：企业 VPN 流量（已知 VPN 服务器 IP）
- **CDN 加速**：Cloudflare/阿里云 CDN 的正常 DNS 解析
- **IPv6 隧道**：6to4/Teredo 等合法 IPv6 隧道

### 判断攻击是否成功
| 隧道类型 | 成功特征 | 检测方法 |
|---------|---------|---------|
| DNS 隧道 | 单域名查询 >100次/min，子域名 >50 字符 | DNS 频率 + 子域名长度 |
| ICMP 隧道 | ICMP 包 payload >64 字节，频率异常 | 包大小 + 频率 |
| HTTP 隧道 | 固定 URL 路径 POST，请求/响应大小相似 | URL 模式 + 请求频率 |
| SSH 隧道 | SSH 会话时间长，数据量远超交互式 Shell | 会话时长 + 流量大小 |

### 关联分析
- **DNS 统计**：同源 IP 的 DNS 查询频率、子域名长度分布、TXT 记录比例
- **流量基线对比**：与该 IP 的历史通信模式对比（频率、大小、方向）
- **前置告警**：隧道通常在 RCE 成功后建立，追溯是否有漏洞利用告警
- **目的 IP**：隧道对端是否为已知恶意 IP 或 IDC 服务器

## 处置流程

1. **紧急**：确认隧道通信 → 立即隔离主机、封禁隧道对端 IP/域名
2. **高**：可疑隧道行为 → 监控、深入分析 DNS/流量模式
3. **中**：异常 DNS 查询 → 封禁域名、持续监控
4. **低**：误报（合法 VPN/CDN） → 白名单
