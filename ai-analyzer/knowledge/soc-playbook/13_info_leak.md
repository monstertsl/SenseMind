# SOC 处置手册: 13_info_leak (信息泄露)

## 适用告警

.git 泄露、.env 泄露、源码泄露、配置文件泄露、密钥泄露、接口文档暴露
MITRE T1552

## 分析要点

### 判断是否误报
- **公开文档**：网站正常的 robots.txt、sitemap.xml
- **API 文档**：开发环境暴露 Swagger，已确认无敏感信息
- **静态资源**：CSS/JS/图片等正常静态文件
- **版本信息**：响应头中的 Server/X-Powered-By（非主动泄露）

### 判断泄露是否成功
| 泄露类型 | 成功标志 | 响应特征 |
|---------|---------|---------|
| .git 仓库 | HTTP 200 + `ref: refs/heads` | 响应体为 Git 配置 |
| .env 文件 | HTTP 200 + `KEY=value` 格式 | 包含 `PASSWORD=`、`API_KEY=` |
| 配置文件 | HTTP 200 + 数据库/应用配置 | 包含 `DB_PASSWORD`、`SECRET` |
| 备份文件 | HTTP 200 + 文件下载 | 响应体为 SQL/ZIP/TAR |
| 密钥文件 | HTTP 200 + `BEGIN RSA PRIVATE KEY` | SSH/AWS 密钥内容 |
| 源码泄露 | HTTP 200 + PHP/Java 源码 | 代码内容 |
| 调试接口 | HTTP 200 + `/actuator/env` 返回环境变量 | Spring Boot 等框架信息 |

### 关联分析
- **同源 IP 请求序列**：攻击者通常批量探测多个泄露路径
- **响应码分布**：大量 404 中穿插 200，命中泄露文件
- **后续利用**：泄露后短时间内是否出现使用该凭据的登录

## 处置流程

1. **紧急**：确认密钥/数据库密码泄露 → 立即轮换凭据、排查是否被利用
2. **高**：配置文件/源码泄露 → 删除泄露文件、修补访问控制
3. **中**：探测泄露路径未成功 → 封禁 IP、检查是否有未发现的泄露
4. **低**：误报（公开文档/静态资源） → 白名单
